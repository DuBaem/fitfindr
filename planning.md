# FitFindr — planning.md
## Tools

### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Searches the mock listings dataset using provided utility functions to filter items by a descriptive keyword, exact size, and maximum price cap.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): Text query representing the item style (e.g., "vintage graphic tee").
- `size` (str): The desired size filter matching the dataset (e.g., "M").
- `max_price` (float): The maximum allowed price.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of dictionaries representing matching listings. Each dictionary contains keys: id, title, description, category, size, condition, price, colors, brand, and platform.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
Returns an empty list []. The agent must catch this, write an error message to the session state, skip all subsequent tools, and prompt the user to broaden their search constraints.

---

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Sends the user's selected item and current wardrobe data to the Groq LLM to generate a stylized, cohesive outfit recommendation.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The selected item dictionary returned from search_listings.
- `wardrobe` (dict): A dictionary containing an 'items' list of the user's current clothing.

**What it returns:**
<!-- Describe the return value -->
A string containing detailed styling advice and outfit assembly instructions.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe input is empty, the agent bypasses the wardrobe matching and generates general, universal styling advice for the new_item alone so the workflow does not crash.

---

### Tool 3: create_fit_card

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Takes the generated outfit suggestion and the original item data, prompting the LLM to write an engaging, social-media-style caption.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (str): The styling recommendation string generated by suggest_outfit.
- 'new_item' (dict): The original selected item dictionary.

**What it returns:**
<!-- Describe the return value -->
A unique string representing an Instagram-style caption.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->
If the outfit input is an empty string, the tool catches the error and returns a predefined fallback text stating the outfit data was lost, preventing a Python exception.

## Planning Loop

**How my agent decides which tool to call next:**
After search_listings runs, the loop explicitly checks if the returned list is empty. If yes, it sets an error message in session["error"] and breaks the loop early, returning to the user. If no, it sets session["selected_item"] = results[0] and proceeds to call suggest_outfit. It then directly passes the output of suggest_outfit into create_fit_card.

## State Management

**How information from one tool gets passed to the next:**
The agent uses a centralized dictionary called session. It stores the output of Tool 1 in session["selected_item"], which is read by Tool 2 and Tool 3. Tool 2's output is saved to session["outfit_suggestion"], which is read by Tool 3. Tool 3 saves to session["fit_card"].

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Agent stops execution, sets session["error"] explaining no items were found, and suggests the user increase their budget or remove the size filter.|
| suggest_outfit | Wardrobe is empty | Agent ignores the missing wardrobe and provides universal styling tips based purely on the newly found item's tags and description. |
| create_fit_card | Outfit input is missing or incomplete | Agent intercepts the empty string, halts the LLM call, and returns a hardcoded error caption instead of crashing the UI. |
